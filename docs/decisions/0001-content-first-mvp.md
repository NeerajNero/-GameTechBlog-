# 0001: Content-First MVP

- Date: 2026-05-19
- Status: accepted

## Context

The project is a gaming + tech blog focused on SEO, social distribution, AdSense readiness, affiliate monetization, and personal brand growth.

The starter pack supports full-stack product builds, but this MVP does not need product infrastructure.

## Decision

Build a static, content-first web MVP.

Do not add backend, database, Prisma, NestJS, Docker, auth, admin panel, CMS, payments, or user accounts in the MVP.

## Consequences

- Lower operational complexity.
- Faster path to publishing articles.
- Content and SEO become the first product surface.
- Future product features can be added only after content traction justifies them.

## Deferred

- CMS.
- Comments.
- Product database.
- Price tracking.
- User accounts.
